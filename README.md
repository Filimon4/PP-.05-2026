### Часть 1. Проектирование информационных систем

Сроки выполнения: 10.03-14.03
Необходимые приложения: Прил_1.rar
1. На основании документов, представленных заказчиком, необходимо спроектировать диаграмму
прецедентов для информационной системы. Предприятие производит продукцию согласно
установленным спецификациям и реализует готовую продукцию заказчикам. Каждая продукция имеет
свою цену, зависящую от стоимости материалов.
Обязательно выделение основных актеров (менеджер по продажам, технолог, экономист, начальник
производства, кладовщик) и ключевых прецедентов (управление справочниками, работа со
спецификациями, расчет себестоимости, оформление заказов, учет производства). При разработке
диаграммы обеспечить согласованную схему именования и использовать отношения включения
(include) и расширения (extend) для отражения логики работы системы.
Диаграмма должна быть представлена в формате .pdf и содержать актеров, прецеденты, связи между
ними и границы системы.
2. На основании документов, представленных заказчиком, необходимо спроектировать ER-диаграмму
для информационной системы. Предприятие производит продукцию согласно установленным
спецификациям и реализует готовую продукцию заказчикам. Каждая продукция имеет свою цену,
зависящую от стоимости материалов.
Обязательна 3 нормальная форма с обеспечением ссылочной целостности. При разработке диаграммы
обратите внимание на согласованную осмысленную схему именования, создайте необходимые
первичные и внешние ключи.
ER-диаграмма должна быть представлена в формате .pdf и содержать таблицы, связи между ними,
атрибуты и ключи (типами данных на данном этапе можно пренебречь).

# Результат: диаграмма прецедентов ф файле usecase.pdf, ER-диаграмма в файле erd.pdf

### Часть 2. Соадминистрирование баз данных и серверов

Сроки выполнения: 16.03-19.03
Необходимые приложения: Прил_1.rar
1. Создайте базу данных на основании разработанной ER-диаграммы, используя предпочтительную
платформу PostgreSQL. Создайте таблицы основных сущностей, атрибуты, отношения и необходимые
ограничения. После создания базы данных требуется импортировать данные из файла
«Заказчики.json».
2. Создайте запрос, позволяющий вычислить полную стоимость заказа покупателя с учетом следующей
информации:
- количество продукции в заказе;
- стоимость всех материалов, использованных для производства данной продукции (учитывая
норму расхода).

# Результат: физическая модель БД (из PostgreSQL), SQL-скрипт запроса.
